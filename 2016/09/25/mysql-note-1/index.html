MySQL从入门到狗带 ——第一话：MySQL入门篇

  • MySQL
  • 学习笔记
  • 数据库

各位观众老爷们大家好，这里是新专题《MySQL从入门到狗带》~(●’◡’●)

珍爱生命，远离MySQL。_(:3」∠❀)_

第一话: MySQL入门篇

基本概念

几种常见关系型数据库：

  1. SqlServer：是微软公司数据库，适用于windows平台，服务器需要付费。
  2. MySql：是Oracle公司数据库，性能不错，社区版免费，其体积小、速度快、总体拥有成本低，且为开源项目，适用平台有windows, linux, unix。
  3. DB2：IBM公司的数据库，大型数据库系统。

吾辈学习的是MySQL~嗯、、贴个简介？
MySQL是一个关系型数据库管理系统，由瑞典 MySQL AB 公司开发，目前属于 Oracle 旗下公司。MySQL是最流行的关系型数据库管理系统，在 WEB 应用方面 MySQL 是最好的 RDBMS (Relational Database Management System，关系数据库管理系统) 应用软件之一。MySQL 是一种关联数据库管理系统，关联数据库将数据保存在不同的表中，而不是将所有数据放在一个大仓库内，这样就增加了速度并提高了灵活性。MySQL 所使用的 SQL 语言是用于访问数据库的最常用标准化语言。MySQL 软件采用了双授权政策，它分为社区版和商业版，由于其体积小、速度快、总体拥有成本低，尤其是开放源码这一特点，一般中小型网站的开发都选择 MySQL 作为网站数据库。由于其社区版的性能卓越，搭配 PHP 和 Apache 可组成良好的开发环境。

MySql数据库的下载和安装

社区版下载地址，你也可以去官网自行挑选~

吾辈遇到的唯一一道槛(或许不是槛，称其为无法翻越的大山更为贴切QAQ)就是在安装过程中Starting Server处卡死。因为MySQL的安装会在很多个文件夹里添加相关文件，有隐藏的有可见的，但是如果出现问题需要重装的话，就需要把之前所有的程序都卸载，所有文件都删除。具体步骤：

  1. 控制面板——>程序和功能——>删除所有安装的mysql程序
  2. 到C:\删除所有相关文件（这里我是在搜索栏直接输MYSQL找的）有些文件是隐藏的，你需要文件夹选项->查看->显示隐藏文件。
  3. 打开命令行窗口（cmd）,输入regedit（注册表编辑器），ctrl+F查找MySQL，如有以下三个文件，统统删掉。(因为吾辈失败了很多次，所以之后在这一步把能查到的所有MySQL相关的东西全删了，但好像有两个无法对其进行更改)
    HKEY_LOCAL_MACHINE/SYSTEM/ControlSet001/Services/Eventlog/Applications/MySQL HKEY_LOCAL_MACHINE/SYSTEM/ControlSet002/Services/Eventlog/Applications/MySQL HKEY_LOCAL_MACHINE/SYSTEM/CurrentControlSet/Services/Eventlog/Applications/MySQL
    至此就删的差不多了，你要明白，如果想重装成功，之前的MySQL一定要清理得干干净净渣都不剩QAQ（因为是重要的话所以此处要x3）

接下来还有几处小细节~

  1. 设置数据库字符集为utf8
  2. 查看数据库字符集：show variables like ‘%char%’;
  3. 修改
    • 打开my.ini文件
    • 添加：default-character-set=utf8 和 character-set-server=utf8
    • 重启服务器

MySql数据库基本操作

  1. 服务器的启动与停止
    在任务栏搜索“服务”，找到MySQL，右击，可以查看服务端状态，MySQL处在启动状态的时候，客户端才能连上。
  2. MySql客户端登录：启动客户端程序后默认root用户登录，输入安装时root的密码，回车。


一个简单的show databases;命令，别忘了分号yooo~

  1. MySQL命令行客户端
命令 作用
show databases; 显示所有数据库
use 数据库名; 切换到指定数据库
show tables; 显示当前数据库中所有表名
select * from 表名; select * from 表名

数据库软件组成

  1. 服务端
    • 存储数据
    • 解析和执行命令
    • 服务端需要保持启动状态
  2. 客户端
    • 首先连接服务器
    • 编写sql命令发送到服务器
    • 客户端软件
      • 1) mysql自带的命令行（Command Line Client）,即可视化客户端
      • 2) 第三方客户端，例如Navicat

Navicat for MySQL

Navicat for MySql是一款非常实用的客户端管理工具，以图形界面的形式来管理MySql数据库。它也是吾辈学习时主要依赖的工具。

  1. 连接服务器
    打开Navicat for MySql，单击左上角连接，输入密码，测试，成功即可连接
    如果不成功，就将localhost改为127.0.0.1:本机的回路地址。
  2. 查看数据库
    • 双击数据库，可以查看数据库包含的对象
    • 查看表 ：展开表可以查看当前数据库中都有哪些表，双击表名可以看表中数据
    • 查看表结构： 右击表-设计表，可以对表中字段重新设计
  3. 创建数据库
    • 右击服务器-新建数据库，输入数据库名，字符集和排序规则选择utf8（不然可能会出现中文无法识别的现象）
    • 创建表：右击表-新建表，打开表设计器，设计表的字段
    • 设计好后保存表

  4. 操作表数据

    • 增加：insert into 表名(形参(字段)列表) values(实参(字段值));按顺序一一对应，varchar类型要用用单引号引起来
    • 删除：delete from 表名 where 条件（如果不加条件，整个表都会被删掉）
    • 修改：update 表名 set 字段名1=值1, 字段名2=值2, …… where 条件

  5. 查询

    • 查询所有：查询所有： select * from 表名
    • 查询指定条件的行： select * from 表名 where 条件
    • 查询某些列： select 字段1, 字段2, ……, from 表名 where 条件

  6. 数据库迁移

    • 数据库脚本导出：右击数据库-转储SQL文件
    • 数据库脚本导入：先建立 同名数据库 ，然后右击数据库（处于打开状态）- 运行SQL文件

END
