Java从入门到放弃 ——第二话：包和修饰符

  • Java

各位观众老爷们大家好，这里是专题系列《Java从入门到放弃》~(●’◡’●)

为您细细道来我是如何弃疗Java的。_(:3」∠❀)_

第二话：包和修饰符

包（Package）

概念

包（package）是Java提供的一种区别类的名字空间的机制，它提供了访问权限和命名的管理机制。

  1. 将功能相近的类放在同一个包中，可以方便查找与使用。
  2. 由于在不同包中可以存在同名类，所以使用包在一定程度上可以避免命名冲突。
  3. 在Java中，某类访问权限是以包为单位的。

JDK常用包介绍

包的应用

  1. 包的声明：
    • 手工开发：在源文件的开始加上：package <包名> (要求全~部~小~写~，反~域~名~制)
    • 在IDE工具下声明包：在Eclipse中通常先创建package,然后在包下创建类

注意：* 一旦声明了包，则类名就是带有完整包名的类

* 包在文件中的表现就是文件夹目录结构，功能相似的类放在同一目录下
* 相当于对类进行了包装，在不同的包中允许有相同类名存在
  1. 包的使用
    • 如果当前类要用到其他包中的类，需要使用import关键字来导入
    • 快捷方式：光标定位使用类，Alt+/
    • 如果需要用到某个包的多个类，可以用”*”代替所有类

访问控制符

为什么要使用访问控制符？信息隐藏（隐藏内部实现细节，提供外部访问接口）是 OOP 最重要的功能之一，也是使用访问修饰符的原因。

JAVA访问控制符

修饰类型 同一个类 同一个包内的类 不同包内的子类 不同包并且不是子类
private(默认) 是 否 否 否
protected 是 是 是 否
public 是 是 是 是

通常用法：属性设置成private类型，方法设置成public类型或protected类型。

非访问控制符

静态static

静态属性或方法属于类属性或类方法，即为整个类所有，不单独为某个对象使用。
静态数据保存在栈中，为所有对象共享内存空间，即只开辟一个存储空间。
使用方法：由类直接调用

应用方式

  1. 修饰方法

    • 可以直接调用其他静态方法
    • 可以直接访问静态数据（属性）
    • 若要访问非静态方法和数据，需要创建对象后访问
    • 不能使用 super 或 this 关键字
    • 特殊：非静态方法可以直接调用静态方法

  2. 修饰属性

    • 静态或非静态方法都可以直接使用

  3. 静态块

    • 类加载后就执行(JVM)

示例代码

final

  1. 定义常量值
  2. 定义方法：子类不能重写方法
  3. 定义类：该类不能被继承

abstract

  1. 定义类
    • 抽象类不能被实例化，即不能new对象
  2. 定义方法
    • 没有方法体
    • 构造方法和static 方法不能是抽象的
