Java从入门到放弃 ——第一话：封装，继承，抽象类与接口

  • Java

第一话：封装，继承，抽象类与接口

封装

概念

1.创建类时设计属性和方法，属性和方法属于该类 ，属性和方法结合为一个整体。
2.尽可能隐藏对象的内部实现细节，只提供外部的访问接口。

封装的实现

1.将需要隐藏的部分用private进行修饰；
2.公共的接口用public修饰；
3.对于java简单类，主要是实现属性私有化和公共的get,set方法。

toString()

官方给出的使用说明(在代码编辑区按下Ctrl，然后光标移到某个自己不明白的函数上，点击即可查看jdk源码)是酱紫的：
Returns a string representation of the object. In general, the{@code toString} method returns a string that “textually represents” this object. The result should be a concise but informative representation that is easy for a person to read.It is recommended that all subclasses override this method.

说白了，toString()是对象的字符串表现形式，是Object类的方法，如果重写了toString(),在打印对象时会自动调用toString()方法。

小结

封装可以说是体现面向对象编程特性的基本和核心。在初学者的练习中，构建类时经常（最起码）会用到的是构造函数，get、set方法和toString()方法。

继承

概念

  1. 所有Java的类均是由java.lang.Object类继承而来的，所以Object是所有类的祖先类。
  2. 继承是面向对象的程序中两个类之间的一种关系，即一个类可以从另一个类（即它的父类）继承属性和行为。被继承的类称为超类（父类），继承父类的类称为子类。
  3. 继承使用extends关键字，例如：

`public class A1 extends A{

}`

表示A1继承了A，A1是A的子类。

意义（为什么要用继承）

  1. 代码复用：子类继承了父类，则继承了父类的公有属性和方法，子类中可以直接调用。
  2. 重写父类方法：将父类的方法重新实现，例如toString();
  3. 扩展子类本身的方法。
  4. 重点是对子类的应用。

小结

重载(overload)和重写(override)

  • 方法重载：多个同名函数同时存在，具有不同的参数个数or类型。

  • 方法重写：父类与子类之间的多态性，对父类的函数进行重新定义。如果在子类中定义某方法与其父类有相同的名称和参数，我们说该方法被重写 (Overriding)。在Java中，子类可继承父类中的方法，而不需要重新编写相同的方法。

抽象类

不确定某个方法的具体实现细节，将方法设置为abstract类型，则类也要设成abstract类型，

创建抽象父类Car

注意

  1. abstract只能修饰类和方法，不能修饰属性，因为不允许属性不确定
    private abstract float weight; 出错

  2. 也不能修饰构造方法:

    • 因为：
    • 1.子类的构造方法无条件调用父类无参构造
    • 2.类通过构造方法创建对象
    • 所以构造方法不能没有实现
      public abstract Car(String brand, String color, float price) { super(); this.brand = brand; this.color = color; this.price = price; }
      因此只能有一个无参构造，这样写会出错。

  3. abstract不能和static连用

    • 因为：static 是静态的，属于类方法，子类也要沿用类方法。
    • 凡是静态的，当类加载到虚拟机（JVM）就直接执行。
      public abstract static void speedUp(); 错错错

  4. abstract不能和private连用

    • 不能对父类私有的方法进行重写，就不能对抽象进行实现
    • 子类能够继承父类的属性和方法，但是不能访问私有成员
      private abstract void speedUp(); 错错错

3 4 会出现的错误提示 The abstract method speedUp in type Car can only set a visibility modifier, one of public or protected

抽象类的特征

  1. 不能被实例化（不能new对象）
  2. 有抽象方法的类一定是抽象类
  3. 抽象类可以没有抽象方法

抽象类的用途

  1. 设计出父类，供子类继承，实现代码重用，同时给子类提供了模板。如果子类也不进行实现，那就将子类也设置成抽象类。
  2. 设计类需要实现的功能（某些功能不具体），由派生类来具体实现。
  3. 为了保护类的数据和方法，将类设为abstract类型，只有派生类才能创建对象。
  4. 如果子类没有实现抽象方法，子类也要设置成abstract类型。

接口（interface）

当所有方法都是抽象的，就把类定义成接口。

  • 接口中定义的方法必须是public abstract（可省略），即所有方法都没有方法体。
  • 接口中定义的属性必须也一定是public static final 类型，即都是常量，不允许修改。
  • public和abstract修饰可以省略，属性public static final 修饰也可省略。

使用接口可以实现多继承，这样就解决了类的单一继承问题

  1. 接口可以多继承——接口之间多重继承
  2. 接口可以多实现——某个类实现多个接口
