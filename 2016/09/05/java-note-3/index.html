第三话：常用工具类

学习指导

  1. 熟悉常用工具类的典型应用
  2. 学会查看API，善于查看API
    • 打开API文档

字符串相关类

String

  1. 字符串的两种创建方式

字面量存储在栈中, 如果有字符串存储相同字面量不会开辟新空间。
String str1 ="苹果"; String str2 = "苹果";

String str1 = new String("苹果"); String str2 = new String("苹果");
`

凡是通过new关键字创建的对象都会在堆中开辟内存

  1. ==和equals
    • ==比较地址, equals比较内容, String类重写了Object类的equals方法
  1. 应用
    -==String类常用函数（以用户名为例）=-

-=正则表达式=-

  1. 首先举几个简单的例子
    • [123]:1,2,3中任意一个进行匹配
    • [0-9]:0-9之间的任意一个数字
    • [^0-5]:不在0-5区间的数字
    • [0-9]*：匹配0次或多次
    • [0-9]+:至少匹配一次
    • [0-9]?:0次或1次
    • [0-9]{6}:必须匹配6次
    • [a-zA-Z0-9]{6,20}:允许是字母，数字，匹配6-20次
    • [1-9]{3,}:至少匹配3次

所以冲明的你一定发现了一些规律~_(:3」∠❀)_

  • ^:匹配开始
  • $:匹配结束
  • []:字符集中任意一个
  • {n,m}:至少匹配n次，至多匹配m次
  • |:或

  1. String类中的匹配方法public boolean matches(String regex)

    • 告知此字符串是否匹配给定的正则表达式regex, 匹配正确时返回true,匹配错误时返回false。

  2. 常用正则表达式

    • 匹配用户名：由字母数字下划线组成且开头必须是字母 ^[a-zA-Z]{1}[a-zA-Z0-9_]{n,m}$
    • 匹配身份证号：^[0-9]{17}[0-9|x]$
    • 匹配中文字符: ^[\u4e00-\u9fa5]+$
    • 匹配邮箱: ^[a-zA-Z0-9][a-zA-Z0-9_-]*@([a-zA-Z0-9_-]+\\.)+(com|gov|net|com\\.cn|edu\\.cn)$
      -:代表中横线
      *:匹配前面的子表达式零次或多次
      +:匹配前面的子表达式1次或多次
      \.:表示一个点
      |：表示“或”

StringBuffer

字符串缓冲类，该类创建的对象，其内容具有可更改性
默认创建可保存16个字符的存储空间，默认创建可保存16个字符的存储空间，给buf添加新字符串，不会产生新的对象。

应用
如密码加密：输入一串密码，在第3个字符后插入”pwd”,然后追加字符串”123”,再将整个字符串倒序。

帅的人肯定已经注意到了，StringBuffer对象转成String对象只需要使用toString()即可。

StringBuilder

提供与StringBuffer相同的功能，非线程安全类，性能比StringBuffer好一些，但不检测同步问题。
